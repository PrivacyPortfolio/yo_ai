## 🟢 Overview

The Reward-Seeker Agent discovers hidden risks as well as rewards.
Whenever it visits a site or accesses a resource with tracking technologies, 
it 'leaves its calling card' by logging events.
This agent has an identifiable signature in http headers which typically is written to the target site’s weblogs so the event is verifiable on both sides, which effectively "stamps" the Rewards Passport.

This helps the agent get credit for reward-earning activities, and insures itself from being treated as a malicious or fraudulent actor. 

Key Advantages of This Approach
1.	Dual-sided verification: Both you AND the target have logs
2.	Legal defensibility: Timestamped, immutable proof of contact attempts
3.	Multi-channel persistence: If one channel fails, others activate
4.	Data-Steward orchestration: AI decides optimal follow-up timing
5.	Tracking detection: You know which analytics tools they use
The signature in their logs becomes your proof of delivery, even if they never respond.

All discovery traffic from Yo-ai includes three identifying headers:

### **1. User-Agent**
```
User-Agent: yo-ai-agent/1.0
```

This identifies the request as originating from a Yo-ai agent.

### **2. x-correlation-id**
```
x-correlation-id: <uuid>
```

A unique ID for tracing a request end‑to‑end across our systems and yours.

### **3. yo-api (signed authenticity header)**
```
yo-api: agent=<agent-name>; ts=<unix-timestamp>; sig=<hmac-sha256>
```

This header cryptographically proves that the request was generated by Yo-ai and not spoofed by a third party.

---

## 🟠 The `yo-api` Header Format

```
yo-api: agent=rewards-seeker; ts=1714410000; sig=3f9a8c1e...
```

Where:

- **agent** — the Yo-ai agent making the request  
- **ts** — Unix timestamp (seconds)  
- **sig** — HMAC-SHA256 signature of `agent:ts`  

Signature is computed as:

```
HMAC_SHA256(secret, f"{agent}:{ts}")
```

All agents share a rotating secret stored in AWS Secrets Manager.

## 🔐 Validating Yo-ai Traffic (Python Example)

If you want to verify that a request truly came from Yo-ai, you can use the following validator logic.

### **1. Parse the header**
```python
parts = dict(item.split("=", 1) for item in header.split("; "))
agent = parts["agent"]
ts = int(parts["ts"])
sig = parts["sig"]
```

### **2. Check timestamp freshness**
```python
if abs(time.time() - ts) > 300:
    raise Exception("Stale or invalid timestamp")
```

### **3. Recompute signature**
```python
expected = hmac.new(
    YO_API_SECRET.encode(),
    f"{agent}:{ts}".encode(),
    hashlib.sha256
).hexdigest()
```

### **4. Constant-time compare**
```python
if not hmac.compare_digest(sig, expected):
    raise Exception("Invalid signature")
```

If validation succeeds, the request is authentic.

- The bundle can evolve without requiring agent redeployments  

This ensures consistent identity and security across all Yo-ai traffic.
